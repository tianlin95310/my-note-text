加载 -- 连接 -- 初始化 -- 使用 -- 卸载 

1，加载
有import时，系统就会提前加载
类的加载的最终产品是位于堆区中的Class对象。

2，卸载
该类所有的实例都已经被回收，也就是java堆中不存在该类的任何实例。
加载该类的ClassLoader已经被回收。
该类对应的java.lang.Class对象没有任何地方被引用，无法在任何地方通过反射访问该类的方法。

3，静态的属性和方法不会随着对象的消亡而消亡，他生命周期是跟随者类的

4，在Android中类被卸载是在进程结束的时候，因为ClassLoader是与进程共存亡的，
所以正常情况下静态变量是一直存在的，但当应用进程被系统回收干掉的时候静态变量的值可能会被清除
